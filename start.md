# MarksCraft (ResultMaker) Startup Guide

This guide outlines the exact steps to completely start your application environment, expose it to the public internet, and ensure background generation works flawlessly.

---

## 1. Prerequisites (Usually Automatic)
Your system is configured to run these core services automatically:
* **Nginx & PHP-FPM:** Serving your Laravel files on Port 80.
* **ONLYOFFICE Server:** Running in a Docker container on Port 8088.

### How to manage ONLYOFFICE:
*(If you ever restart your PC and ONLYOFFICE isn't running, here are the commands you need)*

**To check if it is running:**
```bash
docker ps
```

**To start the existing container (Do this first if it's off):**
```bash
docker start onlyoffice-ds
```

**To completely recreate it from scratch (Only if the container was deleted):**
```bash
docker run -i -t -d -p 8088:80 --name onlyoffice-ds -e JWT_SECRET=super_secret_jwt_key_123_must_be_at_least_32_chars onlyoffice/documentserver
```

---

## 2. Start the Queue Worker (For Marksheet Generation)
To ensure the system can generate hundreds of marksheets in the background without freezing the browser, you must run the Queue Worker.

Open a terminal, navigate to your project, and run:
```bash
php artisan queue:work
nohup php artisan queue:work > storage/logs/queue.log 2>&1 &
```
**⚠️ Important:** Leave this terminal window open. As long as it is running, it will automatically detect and process generation jobs.

---

## 3. Expose the Application via Cloudflare
To make both your Laravel application and the ONLYOFFICE Document Server accessible over the internet, run the automated script.

Open a **second** terminal window and run:
```bash
./expose-website.sh
```

This script will automatically:
1. Clean up old tunnels.
2. Generate a public URL for your Laravel app.
3. Generate a public URL for ONLYOFFICE.
4. Print both URLs clearly to your screen.

---

## 4. Update the Plugin Settings
Because Cloudflare Quick Tunnels generate a random, new URL every time you run them, you need to tell your application where ONLYOFFICE lives today.

1. Copy the **🟠 ONLYOFFICE Server URL** generated by the script above.
2. Log into your MarksCraft web dashboard.
3. Go to your Document Server / Plugin settings page.
4. Paste the new URL into the Document Server URL field and Save.

**You are now fully online!** You can share the MarksCraft App URL with anyone on the internet, and your Visual Editor plugin will load perfectly.

---

## 5. Helpful Commands (Viewing Logs)
If you ever need to troubleshoot why something isn't working, you can view the live logs by running these commands:

* **Watch Queue Worker processing:** `tail -f storage/logs/queue.log`
* **Watch Laravel errors:** `tail -f storage/logs/laravel.log`
* **Watch Cloudflare App traffic:** `tail -f storage/logs/cloudflared-app.log`
* **Watch Cloudflare ONLYOFFICE traffic:** `tail -f storage/logs/cloudflared-onlyoffice.log`
* **Watch Nginx Web traffic:** `tail -f /var/log/nginx/markscraft_access.log`
